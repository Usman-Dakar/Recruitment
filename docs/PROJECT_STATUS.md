## 📝 Decisions Made This Session
- Logo area in Navbar syncs width with sidebar state (`w-[224px]` expanded → `w-8` collapsed) so navbar and sidebar stay visually aligned.
- Sidebar uses `NavLink` with `end: true` for `/home` (exact match), `end: false` for all other routes (matches sub-routes when feature modules are built).
- `ProtectedRoute` uses `<Navigate replace>` (not push) so back-button doesn't loop.
- `AppShell` uses `overflow-hidden` on the root + `overflow-y-auto` on `<main>` so only the content scrolls, not the whole page.
- `exactOptionalPropertyTypes: true` strictness required `end: boolean` (not `end?: boolean`) on NavItem and `checked = false` default on DropdownMenuCheckboxItem.
- Auth multi-step flow managed via `step: 'landing' | 'login' | '2fa'` local state in `LoginPage.tsx` (not Zustand — ephemeral UI-only state that doesn't survive navigation).
- `useVerify2FA` calls `setAuth` in `onSuccess` — keeps store mutation inside the hook, out of the page component.
- Mock login always returns `requires2FA: true` — real API can return `false` to skip 2FA step.
